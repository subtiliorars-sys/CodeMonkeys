## If you're here to build

1. Check the kanban for CodeMonkeys cards in Todo
2. Read `WAVES.md` for wave protocol (one wave = one PR)
3. Read `docs/STATE.md` for current state
4. Read `CLAUDE.md` for conventions
5. Branch: `work/<topic>` or `automation/wave-*`
6. Run `pytest` before claiming done
7. Never merge your own PR
